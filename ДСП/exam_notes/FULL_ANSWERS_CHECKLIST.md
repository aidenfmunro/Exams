# Чеклист полных ответов по билетам ДСП

## Цель

Довести каждый билет до уровня подробного учебного ответа: строгие определения, мотивировка, полные формулировки теорем, доказательства или честные схемы доказательств там, где в программе указано “без доказательства”, примеры, контрпримеры, связи с другими билетами и готовая устная версия.

## Формат работы

Работаем по одному билету за раз. Когда пользователь просит “сделай билет N”, нужно:

1. открыть текущий файл `exam_notes/tickets/NN.md`;
2. сверить тему с `exam_notes/EXAM_PROGRAM.md`;
3. при необходимости использовать источники из `materials/`;
4. переписать билет в расширенном формате;
5. оставить формулы в Obsidian LaTeX через `$...$` и `$$...$$`;
6. если материал уже изучен в предыдущих билетах, ссылаться на них с помощью Obsidian-ссылок `[[ ]]`;
7. отметить билет как выполненный в этом чеклисте.

## Критерий готовности одного билета

- [ ] Есть подробный ответ “как в учебнике”, но адаптированный для устного экзамена.
- [ ] Все определения сформулированы строго.
- [ ] Все основные утверждения сформулированы с условиями.
- [ ] Есть доказательства ключевых утверждений.
- [ ] Если доказательство по программе не требуется, это явно сказано, но дана идея конструкции.
- [ ] Есть 2-4 содержательных примера.
- [ ] Есть типичные контрпримеры или предупреждения, где это полезно.
- [ ] Есть блок “что писать на доске” с формулами.
- [ ] Есть список дополнительных вопросов с ответами.
- [ ] Связи с другими билетами оформлены через Obsidian-ссылки `[[ ]]`.
- [ ] Формулы корректно оформлены для Obsidian.

## Правило отметки

После расширения билета заменить `[ ]` на `[x]` только у соответствующего билета. Если билет частично начат, не отмечать его выполненным.
